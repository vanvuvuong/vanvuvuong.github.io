Big-O trả lời 1 câu hỏi: Khi input tăng, thời gian chạy tăng như thế nào?

MÔ HÌNH RAM - MÁY TÍNH GIẢ ĐỊNH

Để so sánh thuật toán mà không phụ thuộc ngôn ngữ/phần cứng:

Giả địnhThực tế
Mỗi operation (+, -, *, =, if) = 1 stepMultiply chậm hơn add
Mỗi memory access = 1 stepCache hit ≠ disk read
Unlimited memoryCó giới hạn

Tại sao vẫn dùng? Vì nó đủ chính xác để dự đoán hành vi ở phạm vi lớn. Giống như coi Trái Đất phẳng khi xây nhà - sai về lý thuyết, đúng về thực hành.

BIG-O NOTATION - NGÔN NGỮ SO SÁNH

Ba ký hiệu:

Ký hiệuNghĩaSystem analogy
O(g(n))Upper bound - tệ nhất là g(n)“SLA: response < 200ms”
Ω(g(n))Lower bound - tốt nhất cũng phải g(n)“Minimum processing time”
Θ(g(n))Tight bound - đúng là g(n)“Actual average”

Thực tế: 90% thời gian bạn chỉ cần quan tâm O() - worst case.

CÁC ĐỘ PHỨC TẠP CỦA TỪNG LỚP - TỪ NHANH TỚI CHẬM 

O(1) → O(log n) → O(n) → O(n log n) → O(n²) → O(n³) → O(2ⁿ) → O(n!)

Bảng thực tế (1 operation = 1 nanosecond):

nO(log n)O(n)O(n log n)O(n²)O(2ⁿ)
100.003μs0.01μs0.033μs0.1μs1μs
1,0000.01μs1μs10μs1ms∞
1M0.02μs1ms20ms16.7 min∞
1B0.03μs1 sec30 sec31.7 years∞

QUY TẮC TÍNH NHANH

Cộng các hàm: Lấy thằng lớn nhất

O(n³) + O(n²) + O(n) = O(n³)

Nhân các hàm: Nhân với nhau

O(n) × O(log n) = O(n log n)

Bỏ hằng số:

O(3n²) = O(n²)
O(n² + 1000000) = O(n²)

LIÊN KẾT VỚI HOẠT ĐỘNG CỦA HỆ THỐNG

Hoạt độngĐộ phức tạpVí dụ

Binary searchO(log n)DB index lookup
Scan toàn bộO(n)grep file, full table scan
SortO(n log n)ORDER BY không index
Vòng lặp lồngO(n²)Cartesian join
All subsetsO(2ⁿ)Brute-force config combinations

6. CÁCH PHÂN TÍCH NHANH

Nhìn vòng lặp structure:

# O(1) - không vòng lặp
x = arr[0]

# O(n) - 1 vòng lặp qua n items
for i in arr:
    process(i)

# O(n²) - vòng lặp lồng
for i in arr:
    for j in arr:
        compare(i, j)

# O(log n) - chia đôi mỗi lần
while n > 1:
    n = n // 2

# O(n log n) - vòng lặp n lần, mỗi lần làm log n việc
for i in arr:           # n times
    binary_search(i)    # log n each

7. VÍ DỤ HỆ THỐNG THỰC TẾ

Bài toán: Tìm IPs trùng trong tệp log

# Cách 1: O(n²) - compare every pair
for i in range(len(ips)):
    for j in range(i+1, len(ips)):
        if ips[i] == ips[j]:
            duplicates.add(ips[i])

# Cách 2: O(n) - dùng hash set
seen = set()
for ip in ips:
    if ip in seen:      # O(1) lookup
        duplicates.add(ip)
    seen.add(ip)

1M IPs:

  • Cách 1: ~500 tỷ comparisons → chết
  • Cách 2: ~1M operations → xong ngay

8. KHI NÀO ĐỘ PHỨC TẠP LÀ QUAN TRỌNG?

Phạm viO(n²) có vấn đề?
n = 100Không, 10K ops
n = 10,000Có thể, 100M ops
n = 1MChắc chắn, 1T ops

Nguyên tắc chung cho các tập lệnh:

  • n < 1000: Đéo care, viết gì cũng được
  • n < 100K: O(n²) bắt đầu đau
  • n > 1M: Cần O(n log n) hoặc tốt hơn

  • Cách tốt hơn: O(?)
