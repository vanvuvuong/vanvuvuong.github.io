KHÁC GÌ VỚI CHAPTER 7?

Chapter 7 (Graph Traversal)Chapter 8 (Weighted Graphs)
Các cạnh như nhauMỗi cạnh có trọng số (cost, latency, bandwidth…)
Đường ngắn nhất = ít bước nhấtĐường ngắn nhất = tổng trọng số nhỏ nhất
BFS đủ dùngCần thuật toán phức tạp hơn (Dijkstra, Bellman-Ford)

Ví dụ:

ĐườngSố hopTổng latency
A → B → C2 hop5ms + 100ms = 105ms
A → D → E → C3 hop2ms + 2ms + 2ms = 6ms

BFS chọn đường 1 (ít hop). Dijkstra chọn đường 2 (latency thấp hơn).

DIJKSTRA - MÀY ĐÃ DÙNG MỖI NGÀY

Chỗ mày gặpDijkstra làm gì
OSPF routingTính shortest path tree từ mỗi router
Network tracerouteTìm đường đi qua các hop
Load balancer (least latency)Chọn backend có latency thấp nhất
CDN routingChọn edge server gần nhất
Kubernetes service meshIstio/Envoy tính route tối ưu

Thực tế: Mỗi khi OSPF converge, nó chạy Dijkstra để build routing table.

DIJKSTRA HOẠT ĐỘNG NHƯ NÀO (KHÔNG CODE)

Bài toán: Từ router A, tìm đường ngắn nhất tới tất cả routers khác.

BướcHành độngGiống cái gì
1Bắt đầu từ A, đánh dấu khoảng cách = 0A là nguồn
2Tất cả nodes khác, khoảng cách = ∞Chưa biết đường
3Xét node chưa xử lý có khoảng cách nhỏ nhấtGreedy: ưu tiên gần nhất
4Với mỗi hàng xóm của node đó, cập nhật khoảng cách nếu tìm được đường ngắn hơnRelaxation
5Đánh dấu node hiện tại là “đã xử lý”Không xét lại
6Lặp lại bước 3-5 cho đến khi tất cả nodes đã xử lýDone

VÍ DỤ CỤ THỂ - NETWORK ROUTING

Topology:

        10ms
    A -------- B
    |          |
 5ms|          |3ms
    |          |
    C -------- D
        2ms

Tìm đường ngắn nhất từ A tới tất cả nodes:

BướcNode đang xétKhoảng cách hiện tạiCập nhật gì
Init-A=0, B=∞, C=∞, D=∞-
1A (nhỏ nhất = 0)A=0B=10, C=5
2C (nhỏ nhất = 5)A=0, B=10, C=5D=5+2=7
3D (nhỏ nhất = 7)A=0, B=10, C=5, D=7B=min(10, 7+3)=10 (không đổi)
4B (nhỏ nhất = 10)A=0, B=10, C=5, D=7Done

Kết quả:

Từ A tớiĐường điTổng latency
BA → B10ms
CA → C5ms
DA → C → D7ms

SO SÁNH CÁC THUẬT TOÁN TÌM ĐƯỜNG

Thuật toánTrọng sốTrọng số âmĐộ phức tạpDùng khi nào
BFSKhông-O(V + E)Tất cả cạnh bằng nhau
DijkstraCó✗ Không đượcO((V+E) log V)Phổ biến nhất, trọng số dương
Bellman-FordCó✓ ĐượcO(V × E)Có trọng số âm, phát hiện negative cycle
Floyd-WarshallCó✓ ĐượcO(V³)Cần đường ngắn nhất giữa MỌI cặp nodes

DIJKSTRA VS BFS - KHI NÀO DÙNG GÌ

Tình huốngDùng gìLý do
Tìm server ít hop nhấtBFSCác hop như nhau
Tìm server latency thấp nhấtDijkstraLatency mỗi hop khác nhau
OSPF routingDijkstraCost mỗi link khác nhau
Network diameter (hop count)BFSChỉ quan tâm số hop
Least cost pathDijkstraCost khác nhau

LIÊN HỆ VỚI OSPF/BGP

ProtocolThuật toánGiải thích
OSPFDijkstra (SPF)Mỗi router chạy Dijkstra để build shortest path tree
IS-ISDijkstraTương tự OSPF
BGPKhông dùng DijkstraBGP dùng path vector, chọn theo policy, không phải shortest path thuần
RIPBellman-FordDistance vector, đơn giản hơn nhưng chậm converge

OSPF chi tiết hơn:

Bước OSPFLiên quan Dijkstra
1. LSA floodingThu thập topology (đồ thị)
2. Build LSDBXây dựng đồ thị hoàn chỉnh
3. Run SPF (Dijkstra)Tính shortest path tree từ router này
4. Build routing tableKết quả Dijkstra → next hop

MINIMUM SPANNING TREE (MST) - CÂY KHUNG NHỎ NHẤT

Bài toán khác: Nối tất cả nodes với tổng cost thấp nhất, không cần đường ngắn nhất giữa từng cặp.

Shortest Path (Dijkstra)MST (Prim/Kruskal)
Đường đi tối ưu từ 1 nguồnKết nối tất cả với tổng cost tối thiểu
Có thể đi vòngKhông có cycle (là cây)
Dùng cho routingDùng cho network design

Khi nào dùng MST:

Bài toánDùng MST
Thiết kế mạng mới, chi phí cáp thấp nhất✓
Cluster network, kết nối tất cả servers✓
Broadcast/multicast tree✓
